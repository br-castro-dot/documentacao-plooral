Visão Geral da Integração

Seja bem-vindo à documentação base de integração entre as plataformas Plooral e Studion.mx desenvolvido pela DOT Digital Group.

Este documento serve como guia técnico para orientar a equipe de engenharia no consumo dos serviços de gerenciamento de usuários, consulta e matrículas em trilhas de aprendizagem. Todos os endpoints mapeados e listados no menu lateral correspondem estritamente aos escopos e fluxos combinados e acordados no documento técnico inicial do projeto.

URL Base (Host)

Para realizar as requisições de teste e homologação do projeto, utilize o seguinte endereço de servidor:

https://api-studion-sescoop.hom.dotgroup.com.br
📝 Nota Importante sobre as Rotas: A URL base descrita acima deve, obrigatoriamente, anteceder de todos os caminhos relativos dos endpoints listados nesta documentação.

Exemplo prático: Se o endpoint de autenticação está mapeado como /api/login_check, a requisição final contendo o Host completo deverá ser disparada para:
https://api-studion-sescoop.hom.dotgroup.com.br/api/login_check

Fluxo de Integração Sugerido

Para um consumo correto e ordenado dos endpoints, recomendamos seguir a seguinte esteira lógica de desenvolvimento:

  • 1. Autenticação: Realize a chamada enviando as credenciais para obter o Bearer Token (JWT).
  • 2. Identificação do Aluno: Chame o endpoint de Info do Usuário passando o username para capturar o parâmetro uuid interno.
  • 3. Execução das Ações: Utilize o UUID coletado e o Token ativo para matricular o usuário em trilhas, listar históricos ou baixar os certificados correspondentes.

Autenticar Usuário

Gera os tokens de acesso (Access Token) e de atualização (Refresh Token) a partir das credenciais do usuário. Este token será necessário para autenticar as requisições nos demais endpoints protegidos da API.
Este token JWT resultante será usado para autenticação das chamadas dos próximos endpoints apresentados.

POST /api/login_check

Cabeçalhos (Headers)

NomeTipoDescrição
Content-Type String Obrigatório. Valor deve ser application/x-www-form-urlencoded.
⚠️ Atenção: Certifique-se de enviar os dados no formato x-www-form-urlencoded. Enviar como application/json resultará em erro.

Corpo da Requisição

ParâmetroTipoDescrição
_username String Obrigatório. O e-mail de acesso do usuário.
_password String Obrigatório. A senha de acesso do usuário.

Respostas

200 OK (Sucesso)

{
    "token": "eyJhbGciOiJSUzI1NiIsInR5...[JWT Omitido]",
    "refresh_token": "eyJhbGciOiJIUzUxMiIsInR5...[Refresh Token Omitido]"
}

Buscar Usuário por Username

Retorna as informações detalhadas de um usuário cadastrado a partir do seu username.

💡 Nota de Cache: Os dados de retorno deste endpoint passam por um mecanismo de cache interno com duração de até 1 semana. Alterações muito recentes no cadastro do usuário podem ter um leve atraso de sincronização nesta chamada.
🔑 Importante: Esta requisição retorna o campo uuid. O valor deste campo será obrigatório para realizar a matrícula do usuário em uma trilha.
GET /api/users/username/{username}

Cabeçalhos (Headers)

NomeTipoDescrição
Authorization String Obrigatório. Bearer <seu_token_jwt>

Parâmetros de URL

ParâmetroTipoDescrição
username String Obrigatório. O identificador do usuário (Ex: usuario.validacao).

Respostas

200 OK (Sucesso)

{
    "id": 919555,
    "active": true,
    "uuid": "a3a2ecb1-1c5c-4494-9d8a-66905789af09",
    "profiles": [
        { "id": 249, "name": "Admin", "alias": "admin" },
        { "id": 250, "name": "Student", "alias": "student" }
    ],
    "meta": {
        "vinculo_com_o_cooperativismo": 1,
        "telefone": "11999999999"
    }
}

404 Not Found

Retornado caso o usuário não exista no banco de dados.

{
    "message": "user_not_found",
    "code": 404
}

Obter Dados de uma Trilha

Retorna os detalhes de uma trilha de aprendizagem específica. O sistema automaticamente padroniza a carga horária (workload) para ser sempre exibida em minutos no retorno, independente de como foi cadastrada.

GET /api/paths/{path_id}

Cabeçalhos (Headers)

NomeTipoDescrição
Authorization String Obrigatório. Bearer <seu_token_jwt>

Parâmetros de URL & Query

ParâmetroTipoDescrição
path_id (URL) Integer Obrigatório. O ID numérico da trilha (Ex: 3314).
embed[] (Query) Array Opcional. Permite incluir dados extras no retorno. Valores suportados: categories, rating_analysis, user_rating.
applicationUserId (Query) Integer Opcional. Obrigatório caso utilize o embed user_rating para obter a avaliação específica de um usuário.

Respostas

200 OK (Sucesso)

{
    "id": 3314,
    "name": "Gestão Orientada por Dados em Cooperativas",
    "description": "Descubra como transformar dados em decisões...",
    "duration": "P0Y0M45DT0H0M0S",
    "isPublished": true,
    "workload": {
        "type": "i",
        "value": 900 // Sempre convertido para minutos pelo sistema
    },
    "closedSession": false,
    "reenrollmentAllowed": true
}

404 Not Found

{
    "message": "paths_not_found",
    "code": 404
}

Matricular Usuário em uma Trilha

Realiza a inscrição de um usuário específico em uma determinada trilha de aprendizagem utilizando o UUID do usuário. Esta requisição possui regras rígidas de negócio validadas antes da efetivação.

POST /api/paths/{path_id}/enroll

Cabeçalhos (Headers)

NomeTipoDescrição
Content-Type String Obrigatório. application/x-www-form-urlencoded
Authorization String Obrigatório. Bearer <seu_token_jwt>

Parâmetros

ParâmetroLocalDescrição
path_id URL Obrigatório. O ID numérico da trilha (Ex: 367).
uuid Body Obrigatório. O UUID do usuário (obtido no endpoint de Info do Usuário).

Respostas de Sucesso

201 Created

{
    "enrollment_id": 512312,
    "path_id": 367,
    "application_user_id": 7545345
}

Regras de Negócio e Erros Mapeados (Domain Violations)

Se a requisição falhar, verifique a mensagem retornada. O sistema previne a matrícula nos seguintes casos:

  • enrollment_process_already_in_progress: Mecanismo de lock do cache. O usuário já tem uma tentativa de matrícula nesta trilha processando nos últimos 10 minutos.
  • application_user_not_active: O usuário tentado encontra-se inativo/bloqueado no sistema.
  • path_ended: A trilha alvo já foi encerrada e não aceita mais inscrições.
  • paths_enrollment_already_exists: O usuário já possui uma matrícula activa nesta mesma trilha.
  • user_can_not_permitted_on_this_branch / student_not_permitted_enroll_on_this_branch: Validação de polos. O usuário não pertence à mesma ramificação/polo exigida pela trilha.

Listar Matrículas de uma Trilha

Busca e lista todas as matrículas ativas em uma trilha específica, processando metadados de acesso, permissões de rematrícula e prazos calculados de expiração (deadline).

GET /api/paths/{path_id}/enrollments

Cabeçalhos (Headers)

NomeTipoDescrição
Authorization String Obrigatório. Bearer <seu_token_jwt>

Parâmetros (Query String)

O endpoint aceita parâmetros padrão de listagem para paginação e filtros:

  • search: Termo de busca livre.
  • page e limit: Controle de paginação.
  • order: Ordenação de colunas.

Respostas

200 OK (Sucesso)

O retorno injeta dados computados no momento da listagem, como canReEnroll (calculado baseado nas regras da trilha) e enrollDeadline (baseado no primeiro acesso + duração da trilha).

{
    // Estrutura paginada contendo:
    "items": [
        {
            "id": 512312,
            "path_id": 367,
            "application_user_id": 7545345,
            "name": "Nome do Usuário",
            "email": "email@usuario.com",
            "status": {
                "id": 513,
                "name": "Aprovado",
                "alias": "aprovado"
            },
            "canReEnroll": false,
            "enrollDeadline": "2026-12-31T23:59:59+00:00",
            "certificate_issued_at": "2026-05-19T10:00:00+00:00",
            "certificate_downloaded_at": null,
            "groups": []
        }
    ]
}

Download de Certificado

Realiza o download do arquivo do certificado. Dependendo do style (Trilha, Componente, Sala de Conferência, etc), o sistema busca o histórico de aprovação validando o certificate_hash informado.

GET /api/myself/path/certificate/download/{certificate_hash}

Cabeçalhos (Headers)

NomeTipoDescrição
Authorization String Obrigatório. Bearer <seu_token_jwt>
Accept String Opcional. É recomendado enviar application/pdf.

Parâmetros de URL

ParâmetroTipoDescrição
certificate_hash String Obrigatório. O hash/UUID de validação único do certificado gerado no momento da conclusão.

Respostas

200 OK (Sucesso)

A requisição retorna o conteúdo binário do documento em PDF. Se o integrador estiver utilizando JavaScript (Axios/Fetch), é crucial configurar a resposta do HTTP Client para aceitar BLOB (ex: responseType: 'blob').

404 Not Found

Pode ocorrer caso o hash informado não exista, ou caso a camada de histórico não encontre dados atrelados àquele hash.

{
    "message": "hash_not_found",
    "code": 404
}

Exemplo cURL

curl --location 'https://api-studion-sescoop.hom.dotgroup.com.br/api/myself/path/certificate/download/574fbb2f-13fa-49b2-b4b8-4cab80165fea' \
--header 'Accept: application/pdf' \
--header 'Authorization: Bearer SEU_TOKEN_JWT_AQUI'
